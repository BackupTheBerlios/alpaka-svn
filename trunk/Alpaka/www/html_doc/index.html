Alpaka is a MVC (Model/View/Controller) framework for building web applications. It's on very alfa stage. You shouldn't use this module on production environments (probably, you can't, anyway). This, is a "quick & dirty" (very quick and very dirty) introduction.

Alpaka is yet another MVC framework for web applications. It pretends to be flexible, agile, and Perlish. So, with Alpaka "There's more than one way to do it" and "Easy things should be easy and difficult things should be possible".

Actually, Alpaka is the "Controller" part of MVC, you can use several "Views" like Template Toolkit or HTML::Template, and you are neither restricted in the "Model" part. You can use plain old perl objects, something like Class::DBI, etc.

Alpaka structures the control of a web application in "Applications", "Components" and "Actions". An Application has Components, and Components has Actions. The URLs are in the form of http://host/application/component/action.do or http://host/component/action.do

If you are looking for a more stable MVC framework based on perl, check out Maypole from Simon Cozens.

You can download the Alpaka distribution from here.

Note that this module isn't in the CPAN yet. It will be uploaded to the CPAN when reaches a more stable level.

To install, unpack the distribution "a la perl": 

perl Makefile.PL
make
make install

Alpaka supports any mod_perl 1 and 2, and any CGI capable web server.
